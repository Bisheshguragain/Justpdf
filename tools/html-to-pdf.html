Tips for Best Results

  • Use inline CSS or internal <style> tags instead of external stylesheets
  • Embed images as base64 data URIs for better compatibility
  • Set explicit page dimensions if you need specific PDF sizes
  • Use web-safe fonts or include font declarations in your CSS
  • Test your HTML in a browser first to ensure it renders correctly
  • Avoid JavaScript-dependent content as it won't execute in PDF
  • Use print media queries (@media print) for PDF-specific styling
  • Keep layouts simple and avoid overly complex nested elements

Frequently Asked Questions

Can I convert an entire website to PDF?

Yes! You can paste the HTML source code of any webpage. For best results, use the "View Source" feature in your browser to copy the complete HTML, or save the webpage as an HTML file.

Will my CSS styling be preserved in the PDF?

Yes! Inline styles and internal stylesheets (<style> tags) are fully preserved. External CSS files should be converted to inline or internal styles for best results.

Can I include images in my HTML to PDF conversion?

Yes! Images embedded as base64 data URIs work perfectly. For external image URLs, they must be accessible from your browser (not blocked by CORS).

What happens to JavaScript in my HTML?

JavaScript code is not executed during conversion. The PDF captures the static HTML structure and CSS styling only. Make sure your content doesn't rely on JavaScript to display properly.

Can I control the page size and orientation?

The converter uses A4 page size by default. You can use CSS @page rules in your HTML to specify custom page dimensions and orientations for more control.

How do I convert multiple HTML pages into one PDF?

Combine all your HTML content into a single HTML file with page breaks (<div style="page-break-after: always;"></div>), then convert the merged file.

Why do my fonts look different in the PDF?

If you're using custom web fonts not available on your system, they may fall back to default fonts. Use standard web-safe fonts or embed font files as base64 for consistency.

Is there a size limit for HTML files?

There's no strict limit, but very large HTML files (several MB) may take longer to process depending on your device's performance and browser capabilities.
